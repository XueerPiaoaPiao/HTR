HTR测试版使用说明：

1、HTR是基于java语言开发的算法，所以程序的执行需要基于Java虚拟机（JDK），请根据计算机系统类型自行安装JDK。推荐安装JDK1.8及以上版本。
2、运行makefile.sh文件会自动将HTR文件安装到/usr/bin文件夹下。安装完成后，输入：htr(或者htr -h)如果出现帮助文档则表示安装成功。
3、测试版仅输出了不同类型对应笼子的数量。计算后输出的结果在和指定轨迹文件所在的文件夹下，输出文件夹和轨迹文件同名。numOfCages文件夹中为每一帧每种类型笼子的数量，其中0.txt表示第0帧文件中笼子数量。output.cage为几种常用笼子数量的汇总。
4、0-12-0中的三个数字从左到右依次表示笼子中4，5，6元环数量，即0-12-0表示sI型5_12笼子。

Instructions for HTR beta:

1. HTR is an algorithm developed based on Java language, so the execution of the program needs to be based on Java virtual machine (JDK). Please install JDK according to the type of computer system. It is recommended to install jdk1.8 and above.
2. Running the "makefile. sh" file will automatically install the HTR file into the /usr/bin folder. After the installation is completed, enter: htr(or: htr -h). if the help document appears, the installation is successful.
3. The output result after calculation is in the folder where the specified track file is located, and the output folder has the same name as the track file. Numofcages folder is the number of cages of each type in each frame, of which 0 Txt indicates the number of cages in frame 0 file. output. Cage is a summary of the number of several common cages.
4. The three numbers in 0-12-0 represent the number of 4, 5 and 6-membered rings in the cage from left to right, that is, 0-12-0 represents sI type 5_12 cages.
